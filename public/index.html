Built for app developers & AI engineers

Turn touch into structured AI input—and haptics into structured AI output.

TouchAI is a touch interaction language and runtime that lets you map gestures to intents and haptic responses without rebuilding gesture detection on every platform.

Star on GitHub Read the docs Read the spec

The problem

Touch is how people use phones and tablets—but AI still mostly speaks text and voice. Every app reinvents gestures with UIKit, Android, or ad-hoc web listeners. There is no shared vocabulary for what a user meant with their fingers, and no standard way for AI to answer with haptics instead of more UI.

The vision

One grammar for touch in and haptics out. Developers write TouchProgram rules once; models train on versioned TouchEventEnvelope JSONL; devices adapt to humans through AI—not the other way around.

How it works — three layers

Input

Touch Language

Normalized touch streams and gesture tokens (tap, swipe, pinch, two-finger swipe). Session baselines make velocity and pressure comparable within a session.

Output

Haptics

Semantic haptics—ack, confirm, clarify—compile to neutral pulse programs, then map to Core Haptics, Android Vibrator, or the web Vibration API.

Infrastructure

Programs & envelopes

Declarative TouchProgram rules plus versioned JSONL envelopes for training, telemetry, and cross-device replay.

Install

The installable core is @touchai/touch-runtime—gesture segmentation, rule evaluation, and haptic materialization. Pair with @touchai/touch-spec for types and schemas. 

npm install @touchai/touch-runtime @touchai/touch-spec
import { segmentGestureFromStream, evaluateProgram } from "@touchai/touch-runtime";
import { makeEnvelope } from "@touchai/touch-dataset";

const gesture = segmentGestureFromStream(touchSamples);
const rule = gesture ? evaluateProgram(program, gesture) : undefined;
const envelope = makeEnvelope({ sessionId: "sess-1", stream: touchSamples, gesture });
